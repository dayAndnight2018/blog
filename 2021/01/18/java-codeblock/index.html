代码块的执行顺序

静态代码块：只执行一次，在类加载到内存时执行一次。

实例代码块：在类创建对象时执行，执行顺序按代码定义的先后顺序依次执行，优先于一般代码块。

一般代码块：方法内的代码块，按定义顺序依次执行。

由于main方法在CodeBlock03中，因此在执行入口方法前会加载这个类，静态代码块会先执行一次。

因此，静态代码块只会执行一次，即类加载的之前执行。后续只会执行实例代码块和构造器，其中实例代码块又是优先执行的。

父类和子类的代码块执行顺序

执行结果如下：

在首次创建子类对象的时候，因为涉及到类的加载，因此会先调用父类静态代码块，然后执行子类静态代码块。

然后依次调用父类的实例代码块、父类的构造器，子类的实例代码块、子类的构造器。

父类创建对象时，不涉及子类的代码执行。
