Stream

流就是对数据的管道式操作

各个步骤的数据源来自于上一个步骤，像一截截管道一样，接续到一起。

  1. filter操作，对源数据进行过滤，选择符合条件的元素。

  2. map操作，将源数据映射到新的数据格式。

  3. limit操作，选取指定数量的元素，而非所有。

  4. collect操作，将流数据转换到集合。

每个流，你只能消费一次

每个流只能使用一次，不可以重复消费

操作流需要什么？

  1. 数据源。对谁操作？

  2. 中间操作。例如筛选，排序。

  3. 终止操作。例如forEach，collect

常见的中间操作

操作 类型 返回类型 参数类型 方法描述
filter 中间操作 Stream Predicate T -> boolean
map 中间操作 Stream Function<T,R> T -> R
limit 中间操作 Stream T -> boolean
sorted 中间操作 Stream Comparator (T,T) -> int
distinct 中间操作 Stream

常见的终止操作

操作 类型 备注
forEach 终止操作 对每一个元素执行对应的lambda操作，返回值为void
count 终止操作 返回流中元素的个数，返回类型为long
collect 终止操作 将流转换成List、Map、Set或者Integer等类型
