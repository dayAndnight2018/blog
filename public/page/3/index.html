占用8位，也就是2^8 = 256个数值,前128个数值为-128——-1，0占一个，剩余127个为正数。所以它的范围是-128——127

在选择数据类型的时候，要尽可能的小，因为大范围的数值类型分配的空间更大，会造成内存的浪费。

byte, short, int, long用于存储整数，float和double用于存储小数。

char 类型允许存储一个Unicode字符

Boolean用于存储两种结果：true 或者 false

变量

在Java中存在两种变量：

  1. 引用变量：存储的是对象的引用

  2. 基本变量：存储的基本数据类型

变量除了具有指定的数据类型外，还具有名称（标识符），其命名规则必须遵循以下几种要求：

  1. 变量名称可以是有限长度的字母、数字、下划线、美元符号组成，需要以字母或下划线开头。

  2. 变量名称不得与关键字重复

  3. 变量名符合作用域要求。

java01 准备工作

一次编写代码，到处运行。

java代码是跨平台的，其实现跨平台的方式通过不同的平台上的虚拟机实现的。

第一行java代码


class MyFirstJava{
    public static void main(String[] args){
        System.out.println("Hello Java!");
    }
}

注意，类名和文件要一致，文件扩展名必须是java

编译你的代码

  1. 在MyFirstJava.java文件所在的目录打开控制台

  2. 执行命令 javac MyFirstJava.java

执行成功后，在相同的目录下会生成类文件：MyFirstJava.class

运行你的代码

  1. 在MyFirstJava.class文件所在的目录打开控制台

  2. 执行命令 java MyFirstJava

执行成功后控制台输出Hello java

当然，你也可以带着参数运行程序：

arg-1代表第一个参数，arg-2代表第二个参数

Git常用指令

基础指令

初始化仓库


    git init

添加文件到缓冲区


    git add filename

添加所有文件到缓冲区


    git add .
    git add --all

清除缓冲区


    git rm -r --cached

删除文件


    git rm filename

提交备注


    git comment -m '备注内容'

注意：git add或git comment失败将不会提交。所以，comment必须有备注内容

查看状态


    git status

查看差异


    git diff filename

版本回退


    git reset
    git reset --hard HEAD^
    git reset --hard version

日志查看


    git log
    git reflog

分支管理

查看当前分支


    git branch

创建分支


    git branch branchName

切换分支


    git checkout branchName

创建并切换分支


    git checkout -b branchName

删除分支


    git branch -d branchName

合并某分支到当前分支


    git merge branchName

标签操作

创建标签


    git tag tagName version

查看标签


    git tag

查看标签详情


    git show tagName

推送某个tag


    git push origin v1.0

推送所有tag


    git push origin --tags

远端库相关

添加远端库


    git remote add origin git://xxx/abc.git

移除远端库


    git remote remove origin git://xxx/abc.git

推送到远端


    git push -u origin master

更新远端的内容到本地


    git pull

组合提交指令


    git rm -r --cached
    git add .
    git comment -m 'update'
    git push origin master
