阿里巴巴开发手册04 OOP约定

OOP约定

  1. 对于静态变量，要求直接使用类名调用

  2. 方法重写需要添加@Override注解

  3. 避免使用可变参数和object类型参数接收

  4. 对于被其他地方使用的接口，禁止变更方法签名，如有需要，使用@Deprecated注解注释

  5. 禁止使用过时的类或方法

  6. Object的equals方法容易抛出异常，建议使用常量的equals方法

  1. 推荐使用equals方法判断包装类对象的值

  2. 基本数据类型与包装类：

POJO类的属性，使用包装类型

RPC的方法参数与返回值使用包装类型

局部变量使用基本数据类型

  1. 对于DO、DTO、VO等POJO类，其定义不要设置属性的默认值

  2. 序列号新增属性时，不要修改序列化ID

  3. 构造函数中禁止加入业务逻辑，如有必要，放到init方法中

  4. POJO类必须带有toString方法

  5. 使用String的split方法时，注意避免分隔符为空的时候，划分结果的问题。

  1. 包含多个构造函数时，按顺序放置在一起，便于阅读，多个同名方法也是

  2. 类内方法的定义顺序，公有方法或保护方法->私有方法->getter或setter方法

  3. setter方法参数建议使用与成员变量相同的名称，不要添加业务逻辑

  1. 循环内的字符串连接，要求使用StringBuilder的append方法

  2. final 的使用

不允许继承的类，使用final定义

不允许修改引用的域对象，使用final定义

不允许重写的方法，使用final定义

不允许重新赋值的局部变量，使用final定义

避免上下文重复使用一个变量，使用final可以强制重新定义一个变量

  1. 谨慎使用Object类的clone方法

  2. 类成员与方法访问控制从严

如果不允许外部直接new对象，构造方法需要使用private

工具类禁止出现public或default构造方法

非静态方法与子类共享，必须是protected

非静态方法本类使用，必须是private

静态变量在本类使用，必须是private

如果是静态变量，请考虑final

仅供本类使用的成员方法，必须private

与子类共享的成员方法，必须protected

阿里巴巴手册03 代码格式

代码格式

  1. 大括号使用要求：

左大括号左侧不换行，需要有一个空格，左大括号后换行

右大括号前换行，右大括号后视情况而定，若是终止符号，则换行，若后面仍有else等代码则不换行。

  1. 左小括号后和右小括号前不空格。

  2. if、for、while、switch、do等保留字与小括号之间必须加空格

  3. 二目及三目运算符左右需要加空格

  4. 采用四个空格进行缩进，禁止tab字符

  5. 注释的双斜线与注释内容之间有且仅有一个空格

  6. 单行字符不超过120个字符，超出需要换行：

第二行相对第一行缩进4个空格，第三行开始不再缩进。

运算符、点符号与下文一起换行

方法的参数中换行，在逗号后进行

括号不要换行

  1. 方法多个参数的逗号后加空格

  2. 默认使用UTF-8编码方式，换行符使用Unix格式

  3. 建议单个方法的总行数不要超过80行

  4. 不同逻辑、不同语义、不同业务代码之间插一个空行隔开，提升可读性。

阿里巴巴开发手册02 常量定义

常量定义

  1. 不允许出现魔法值，直接作为常量使用

  1. long类型的值定义时，需要使用大写L

  2. 对常亮要求使用多个常量类分别管理

  3. 常量的复用

跨应用复用：常量放置在两方的类库中

应用内共享：放置在某类库中

子工程内部共享：放在在子工程的constant目录下

包内共享：放置在当前包的constant目录下

类内共享：类内部使用private static final 定义

  1. 若变量范围较小，包含固定的几个值，如第一季度，第二季度，第三季度，第四季度，则使用枚举

阿里巴巴开发手册01 命名风格

命名风格

  1. 禁止命名以_或$开始或结束

  2. 禁用拼音、中文命名方式

  3. 类名需要首字母大写，驼峰命名。除DO、BO、DTO、VO、AO、PO、UID之外

  4. 方法名称、参数名称、成员变量、局部变量需要统一使用驼峰命名。

  5. 常量需要用全大写，单词之间用下划线连接

  6. 抽象类需要以Abstract开头，异常以Exception结尾，测试类以测试类名称开始，以Test结尾

  7. 数组符号与类型需要紧挨着，如：int[] array

  8. POJO类中的布尔类型，禁止以is开头

  9. 包名需要保持小写，单数语义，类名可以使用复数

  10. 禁止不规范的缩写

  11. 需要使用完整的单词组合

  12. 模块、接口、类、方法使用设计模式，需要体现出设计模式：OrderFactory、LoginProxy、ResourceObserver

  13. 接口类方法上不要添加修饰，避免在接口中定义变量。

  14. Service或DAO接口，要求使用xxxService、xxxServiceImpl命名接口和实现类。普通接口，以形容词为接口名称，Translatable接口或AbstractTranslator实现类

  15. 枚举要求使用Enum后缀，枚举成员用大写，下划线连接。

  16. Service/DAO层命名要求：

获取单个对象用get前缀

获取多个对象用list做前缀，复数形式结尾如：listObjects

获取数量count做前缀

插入用insert做前缀

删除用delete做前缀

修改用update做前缀

  1. 领域数据模型命名要求：

数据对象使用DO作后缀

数据传输对象以DTO做后缀

视图对象以VO做后缀

Git常用指令

基础指令

初始化仓库


    git init

添加文件到缓冲区


    git add filename

添加所有文件到缓冲区


    git add .
    git add --all

清除缓冲区


    git rm -r --cached

删除文件


    git rm filename

提交备注


    git comment -m '备注内容'

注意：git add或git comment失败将不会提交。所以，comment必须有备注内容

查看状态


    git status

查看差异


    git diff filename

版本回退


    git reset
    git reset --hard HEAD^
    git reset --hard version

日志查看


    git log
    git reflog

分支管理

查看当前分支


    git branch

创建分支


    git branch branchName

切换分支


    git checkout branchName

创建并切换分支


    git checkout -b branchName

删除分支


    git branch -d branchName

合并某分支到当前分支


    git merge branchName

标签操作

创建标签


    git tag tagName version

查看标签


    git tag

查看标签详情


    git show tagName

推送某个tag


    git push origin v1.0

推送所有tag


    git push origin --tags

远端库相关

添加远端库


    git remote add origin git://xxx/abc.git

移除远端库


    git remote remove origin git://xxx/abc.git

推送到远端


    git push -u origin master

更新远端的内容到本地


    git pull

组合提交指令


    git rm -r --cached
    git add .
    git comment -m 'update'
    git push origin master

Spring MVC开发环境配置

配置JAVA

  1. 从JAVA开发环境下载地址下载JDK开发环境

  2. 如图所示，安装JDK，在JDK中已经包含JRE

  1. 将JDK路径配置到环境变量
  1. 测试JDK

     java -version

Maven

  1. 从Maven下载地址下载Maven安装包

  2. 解压后，添加环境变量

  1. 测试Maven

     mvn -version

  1. 从此处下载安装包

  2. 一直下一步，安装即可。

  2. 一直next安装即可。在某个步骤需要配置JDK路径。

  1. 打开STS，找到window-preferences-Maven-Installations添加Maven的安装路径到STS
